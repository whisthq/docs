Detailed Description

Copyright (c) 2021-2022 Whist Technologies, Inc.

This file contains all code that interacts directly with receiving and

processing video packets on the client.

Usage

NOTE: Only ONE thread can call a function on video_context at a time, unless a function says otherwise.

VideoContext* video_context = init_video();

In one thread: receive_video(video_context, packet_1); receive_video(video_context, packet_2); receive_video(video_context, packet_3);

In another thread: (NOTE: ONLY render_video is thread-safe. The rest of the functions are not) render_video(video_context); render_video(video_context); render_video(video_context); render_video(video_context);

~~ Join both threads..

destroy_video(video_context);

bool has_video_rendered_yet ( VideoContext *  video_context)

Returns whether or not a video frame has been rendered.

(NOT including any "loading animation" frames) This can be false even after calling render_video, if there simply weren't enough packets yet or if the video is still being decoded.

Parameters
video_contextThe video context that has potentially rendered a frame
Returns
Returns true iff a video frame has actually been rendered out to the SDL framebuffer. (NOT including any "loading animation" frames)
