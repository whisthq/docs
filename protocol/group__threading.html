Detailed Description

Whist API for cross-platform thread management.

This is the Whist API for cross-platform thread creation, destruction and management, along with synchronisation between threads.

◆ WhistThreadPriority

enum WhistThreadPriority

Enumeration of thread priority values.

Enumerator
WHIST_THREAD_PRIORITY_LOW 

Low priority.

Threads with this priority will generally run after other threads, but should still get some time to run.

WHIST_THREAD_PRIORITY_NORMAL 

Normal priority.

This is the default for newly-created threads.

WHIST_THREAD_PRIORITY_HIGH 

High priority.

Threads with this priority will generally run before other threads, but all will still get some chance to run.

WHIST_THREAD_PRIORITY_REALTIME 

Real-time priority.

Threads with this priority will always run first if they can. Do not use this for threads which will not spend a large proportion of time sleeping, as it can prevent other threads (and processes) from running indefinitely.

◆ whist_wait_cond()

void whist_wait_cond ( WhistCondition  cond,
WhistMutex  mutex 
)

Wait for a condition to change.

This atomically unlocks the mutex and starts waiting for a change in the condition to the signalled. When it receives such a signal, it re-locks the mutex and returns.

It can also return at any time for any reason - it must be called in a loop which checks whether the condition has actually changed.

Parameters
condCondition variable to wait for.
mutexMutex protecting the condition, which must be held by the calling thread on entry.

◆ whist_wait_semaphore()

void whist_wait_semaphore ( WhistSemaphore  semaphore)

Wait for a semaphore.

If the semaphore value is greater than zero then decrement it and return immediately. Otherwise, wait until another thread posts the semaphore to increase its value and then decrement it and return.

Parameters
semaphoreSemaphore to wait for.
