anonymous enum

These events are generated and consumed internally by the SDL frontend, typically to manage thread-safety.

All of these events are transmitted using the internal_event_id type. This event identifier is placed in the SDL_UserEvent.code field.

Enumerator
SDL_FRONTEND_EVENT_FILE_DRAG 

File drag event.

data1 contains a pointer to a FrontendFileDragEvent, which the receiver will need to free after use.

SDL_FRONTEND_EVENT_FULLSCREEN 

Fullscreen event.

data1 contains a boolean cast to a pointer indicating whether we want to enter (true) or exit (false) fullscreen mode. data2 contains an int cast to a pointer indicating the ID of the window to fullscreen.

SDL_FRONTEND_EVENT_WINDOW_TITLE_CHANGE 

Window title change event.

data1 contains a pointer to a string, which the receiver will need to free after tuse. data2 contains an int cast to a pointer indicating the ID of the window to change

SDL_FRONTEND_EVENT_TITLE_BAR_COLOR_CHANGE 

Window title bar colour change.

data1 contains a pointer to a WhistRGBColor object, which the receiver will need to free after use. data2 contains an int cast to a pointer indicating the ID of the window to change

SDL_FRONTEND_EVENT_STDIN_EVENT 

Stdin-generated event from stdin (key?value\n).

data1 contains a pointer to a key string (char*), and data2 contains a pointer to a value string (char*) or NULL.

SDL_FRONTEND_EVENT_NOTIFICATION_CALLBACK 

A notification was activated by the user.

Contains no data. Future: indicate which notification it was.

SDL_FRONTEND_EVENT_INTERRUPT 

Internal interrupt from whist_frontend_interrupt().
