◆ try_nacking()

bool try_nacking ( RingBuffer *  ring_buffer,
double  latency 
)

If any packets are still missing, and it's been too long, try nacking for them.

Ideally, this gets called quite rapidly, it has internal timers to throttle nacks. The more rapidly the better, just need to balance CPU usage, 5-10ms should be fine.

Parameters
ring_bufferThe ring buffer to try nacking with
latencyThe round-trip latency of the connection. Helpful with nacking logic
Returns
True if nacking succeded, False if we've bandwidth saturated our ability to nack.
