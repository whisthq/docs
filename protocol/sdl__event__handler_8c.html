This file contains client-specific wrappers to low-level network

functions.

Usage

handleSDLEvent() must be called on any SDL event that occurs. Any action trigged an SDL event must be triggered in sdl_event_handler.c

Function Documentation

◆ sdl_handle_events()

bool sdl_handle_events ( WhistFrontend *  frontend)

Flushes the SDL event queue, and handles all of the pending events.

Calling this function repeatedly is necessary, otherwise "Application not responding" or a Mac beachball may appear.

Returns
True on success, False on failure
Note
This function call MAY occasionally hang for an indefinitely long time. To see an example of this, simply drag the window on MSWindows, or hold down the minimize button on Mac. To see more information on this issue, and my related rant, go to https://github.com/libsdl-org/SDL/issues/1059

◆ sdl_pending_audio_device_update()

bool sdl_pending_audio_device_update ( void  )

The function will let you know if an audio device has been plugged in or unplugged since the last time you called this function.

Returns
True if there's been an audio device update since the last time you called this function, False otherwise.
Note
This function is thread-safe with any other sdl function call.
